# 可用功率下钻页面接口及实现计划

## 背景
可用功率指标需要实现日报表下钻（小时级）和分钟级下钻（秒级）功能，支持用户查看详细的每分钟限电状态、电量数据，以及每秒的功率数据。

## 需求分析
根据需求文档3.5.6：
- **日报表下钻**：24小时×60分钟，每分钟显示限电/非限电、实际电量、可用电量
- **分钟级下钻**：每秒显示实际功率、可用功率
- 需要展示描述信息（非限电电量汇总、AGC指令信息）
- 需要导出功能

## 技术方案

### 数据来源决策：创建详情表
**理由**：
1. 限电判断逻辑复杂（AGC目标值与装机容量比较），不应重复执行
2. 日报表详情是常用功能，查询性能重要
3. 确保与考核结果数据源一致

**详情表存储策略**：
- **分钟级数据**：存入详情表（每天1440条，可接受）
- **秒级数据**：分钟级下钻时查询TDengine（60秒×2字段，数据量小）

### 数据库设计

#### 详情表：t_business_available_power_detail
```sql
CREATE TABLE t_business_available_power_detail (
    id VARCHAR(32) PRIMARY KEY,              -- 随机字符串主键
    config_id BIGINT NOT NULL,
    day DATE NOT NULL,
    time_min VARCHAR(5) NOT NULL,           -- HH:mm
    is_limited BOOLEAN NOT NULL,
    actual_energy NUMERIC(15,4),             -- KWh
    available_energy NUMERIC(15,4),           -- KWh
    agc_target_time TIMESTAMP,
    agc_target_value NUMERIC(10,2),          -- MW
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_config_day ON t_business_available_power_detail(config_id, day);
```

## 接口设计

### 1. 日报表描述接口
```
POST /availablePower/detail/day/description
```
**请求参数**：
```json
{
  "configId": 1001,
  "day": "2024-10-11"
}
```
**返回**：
```json
{
  "description": "2024-10-11日非限电实际电量：1234.5678 KWh，非限电可用电量：1300.1234 KWh；限电时段不参与可用功率准确率计算。"
}
```

### 2. 日报表详情接口
```
POST /availablePower/detail/day
```
**请求参数**：同上
**返回**：
```json
{
  "data": [
    {
      "hour": "00",
      "minutes": [
        {"minute": "00", "isLimited": false, "actualEnergy": "0.1234", "availableEnergy": "0.1300"},
        {"minute": "01", "isLimited": true,  "actualEnergy": "0.0000", "availableEnergy": "0.0000"},
        ...
      ]
    },
    ...
  ]
}
```

### 3. 日报表导出接口
```
POST /availablePower/detail/day/export
```

### 4. 分钟级描述接口
```
POST /availablePower/detail/minute/description
```
**请求参数**：
```json
{
  "configId": 1001,
  "day": "2024-10-11",
  "time": "00:01"
}
```
**返回**：
```json
{
  "description": "AGC指令下达时间2026-09-23 00:00:00.123，指令值为11MW，存在限电。"
}
```

### 5. 分钟级详情接口
```
POST /availablePower/detail/minute
```
**请求参数**：同上
**返回**：
```json
{
  "data": [
    {"second": "00", "actualPower": "10.2345", "availablePower": "12.3456"},
    {"second": "01", "actualPower": "10.2346", "availablePower": "12.3457"},
    ...
  ]
}
```

### 6. 分钟级导出接口
```
POST /availablePower/detail/minute/export
```

## 实现计划

### 阶段1：数据库层
1. 创建实体类 `TBusinessAvailablePowerDetail`
2. 创建Mapper接口 `TBusinessAvailablePowerDetailMapper`
3. 创建Mapper XML文件
4. 执行建表SQL

### 阶段2：修改Task计算逻辑
1. 修改 `XJAvailablePowerTask`，增加明细数据计算
2. 在计算每分钟时，记录限电状态、实际电量、可用电量
3. 记录限电时的AGC指令下达时间和指令值
4. 计算完成后批量插入详情表

### 阶段3：Service层
1. 创建 `AvailablePowerDetailService` 接口
   - `getDayDescription(configId, day)` - 获取日报表描述
   - `getDayDetail(configId, day)` - 获取日报表详情（小时级）
   - `getMinuteDescription(configId, day, time)` - 获取分钟级描述
   - `getMinuteDetail(configId, day, time)` - 获取分钟级详情（秒级，查TDengine）
   - `exportDayDetail(...)`
   - `exportMinuteDetail(...)`

2. 创建实现类 `AvailablePowerDetailServiceImpl`

### 阶段4：Controller层
1. 创建 `AvailablePowerDetailController`
2. 实现6个接口

### 阶段5：VO和导出
1. 创建响应VO类
2. 创建导出VO类

## 测试用例

### 1. Task计算测试
| 用例 | 预期结果 |
|------|---------|
| 全天不限电（AGC目标值≥装机容量） | 详情表1440条记录，is_limited均为false |
| 全天限电（AGC目标值<装机容量） | 详情表1440条记录，is_limited均为true，电量均为0 |
| 部分时段限电 | 对应分钟is_limited=true，电量=0；非限电时段电量正常 |
| 无遥测数据 | 详情表有记录，但电量均为0 |

### 2. 日报表详情接口测试
| 用例 | 预期结果 |
|------|---------|
| 查询正常日期 | 返回24小时数据，每小时60分钟 |
| 查询无数据日期 | 返回空列表或描述信息为0 |
| 查询configId不存在 | 抛出异常 |

### 3. 分钟级详情接口测试
| 用例 | 预期结果 |
|------|---------|
| 查询限电分钟 | 返回60秒数据，description显示限电信息 |
| 查询非限电分钟 | 返回60秒数据，description显示无限电 |
| 查询无TDengine数据 | 返回空数据，功率均为0 |

### 4. 描述接口测试
| 用例 | 预期结果 |
|------|---------|
| 全天非限电 | description显示非限电实际/可用电量 |
| 全天限电 | description显示电量为0 |
| 限电分钟 | description显示AGC指令时间和值 |

### 5. 导出功能测试
| 用例 | 预期结果 |
|------|---------|
| 导出日报表详情 | Excel包含1440行数据，按小时+分钟排序 |
| 导出分钟级详情 | Excel包含60行数据，按秒排序 |
| 导出无数据 | Excel空或包含0值 |

### 6. 数据一致性测试
| 用例 | 验证点 |
|------|--------|
| 限电分钟电量累加 | detail汇总的电量 = 详情表中非限电分钟电量之和 |
| 考核分计算 | 考核分与Task计算的值一致 |
| 限电时间分布 | 详情表中的限电时间与Task判断逻辑一致 |
