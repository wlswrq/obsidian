# 可用功率指标 - 修改与优化记录

## 修改概述

**时间**: 2026-04-20
**目标**: 修改新疆_可用功率指标的遥调取数逻辑，实现性能优化

---

## 一、优化背景

### 原有问题
1. 查询数据量过大（86,400点/天 vs 1,440点/天）
2. SQL层面按值班链路分割，性能低下
3. 同一秒内可能出现AGC目标值突变，导致频繁切流

### 优化目标
- 减少98.3%的数据查询量
- 消除值班链路划分
- 确保同一分钟内AGC目标值一致

---

## 二、主要改动

### 1. 取消值班链路划分
遥调数据查询不再按值班链路分割

### 2. 分钟级数据
查询粒度从秒级改为分钟级

### 3. 前值填充
缺失分钟自动使用上一分钟值

---

## 三、技术方案

### 方案对比

| 方面 | 修改前 | 修改后 |
|------|--------|--------|
| **查询粒度** | 秒级（1秒） | 分钟级（60秒） |
| **数据量** | 86,400点/天 | 1,440点/天 |
| **SQL方式** | UNION ALL按链路分割 | 单一查询 |
| **AGC值处理** | 每秒单独获取 | 同分钟统一值 |
| **缺失值** | 无自动填充 | FILL(PREV)自动填充 |

### 修改文件
- **平台层**：`AdjustmentDataQueryService` 及实现
- **新疆模块**：`XJAvailablePowerTask.java`

## 四、技术实现

### 4.1 代码对比

#### 修改前
```java
// 按值班链路查询（多个UNION ALL）
adjustmentData = adjustmentDataQueryService.queryAdjustmentDataRaw(
    agcCode, startTime, endTime, agcTargetPoints, 1); // 秒级查询
```

#### 修改后
```java

adjustmentData = adjustmentDataQueryService.queryAdjustmentDataRawNoChannel(
    agcTargetPoints, startTime, endTime, 60); // 分钟级查询
```

### 4.2 SQL结构变化

#### 修改前SQL结构
```xml
<!-- 每个值班链路单独查询 -->
<foreach collection="channels" item="channel" separator=" UNION ALL ">
    SELECT
        _wstart AS dataTime,
        ...
    FROM adjustment
    WHERE src_ip = #{channel.srcIp}
      AND src_port = #{channel.srcPort}
      AND dest_ip = #{channel.destIp}
    ...
</foreach>
```

#### 修改后SQL结构
```xml
<!-- 直接按点表号查询 -->
SELECT
    _wstart AS dataTime,
    point_table_number AS pointTableNumber,
    ...
FROM adjustment
WHERE point_table_number IN <foreach>...</foreach>
PARTITION BY point_table_number
INTERVAL(60s) FILL(PREV)
```

### 4.3 值获取逻辑优化

#### 修改前（每秒独立）
```java
for (String secondKey : secondKeys) {
    BigDecimal agcValue = getFloorValue(agcTreeMap, secondKey);
    // 每秒都可能有不同值
}
```

#### 修改后（同分钟统一）
```java
for (String secondKey : secondKeys) {
    String minuteKey = secondKey.substring(0, 16) + ":00";
    BigDecimal agcValue = getMinuteAgcValue(agcTreeMap, minuteKey);
    // 同一分钟内所有秒使用相同值
}
```

## 六、关键技术点

### 6.1 TDengine语法

```sql
INTERVAL(60s) FILL(PREV)
```

- `INTERVAL(60s)` - 60秒为一个时间窗口
- `FILL(PREV)` - 自动使用前一窗口值填充

### 6.2 getMinuteAgcValue()方法

```java
private BigDecimal getMinuteAgcValue(TreeMap<String, BigDecimal> treeMap, String minuteKey) {
    if (treeMap.isEmpty()) {
        return BigDecimal.ZERO;
    }
    Map.Entry<String, BigDecimal> entry = treeMap.floorEntry(minuteKey);
    if (entry == null || entry.getValue() == null) {
        return BigDecimal.ZERO;
    }
    return entry.getValue();
}
```

**作用**：从分钟级数据中获取AGC目标值，支持回溯查找

### 6.3 时间处理

**时间戳格式**：
- 输入：`yyyy-MM-dd HH:mm:ss`
- 内部处理：`yyyy-MM-dd HH:mm:00`
- TDengine返回：`_wstart`（窗口开始时间）
